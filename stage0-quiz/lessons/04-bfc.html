04 · BFC（Block Formatting Context）

这课先把概念读懂，注释填空只是确认你理解了。做布局时用得上，但不如 Flex/Grid 天天写。

是什么（别被英文吓到）

BFC = Block Formatting Context =「块级格式化上下文」。

可以把它想成一个独立的排版小房间：房间里面的盒子怎么摆，有自己的规则； 而且尽量不跟房间外面的布局搅在一起。

为什么要学它（解决什么问题）

  • 父元素里只有浮动（float）子元素时，父级高度塌陷 → 建 BFC 可包住浮动
  • 相邻块的垂直 margin 合并（塌陷）→ BFC 可隔离
  • 两栏：一栏 float，另一栏自适应且不重叠 → 自适应栏建 BFC

现代布局更常用 Flex / Grid。BFC 是「底层原理 + 面试常问」，本阶段能用自己的话讲清即可。

怎么创建 BFC（常见触发，记住几个就行）

  • overflow 不为 visible（如 hidden / auto）
  • float 不为 none
  • position 为 absolute 或 fixed
  • display 为 flow-root / inline-block / flex / grid 等

现在更推荐用 display: flow-root 开 BFC（语义就是「开一个块级格式化上下文」）， 而不是为了清浮动硬写 overflow: hidden（可能裁切内容）。

参考： MDN Block formatting context
