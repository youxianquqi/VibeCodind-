02 · 盒模型与 box-sizing

本课目标：搞清「width 量的是哪一层」，再写全局 border-box。先读完再打开 CSS。

1. 每个元素都是一层盒子

从里到外（记住顺序）：

content（内容）
  → padding（内边距，内容与边框之间）
    → border（边框）
      → margin（外边距，和其他元素的空隙）

margin 在「壳」外面，永远不算进 width。

2. width 有两种算法（只认这两个词）

  • content-box（默认）：你写的 width 只量 content。 再加 padding / border，盒子会变得更宽。
  • border-box：你写的 width 量到 border 外缘。 padding / border 挤在这宽度里面，内容区变小，总宽不变。

严禁误区：没有 width: padding-box 这种写法。 padding-box 是 background-clip 等属性的取值， 不能写在 width 后面。 box-sizing 的合法值主要就是：content-box | border-box。

参考： MDN 盒模型 · box-sizing

4. 为什么要全局 border-box

做侧栏、栅格时，你想的是「这个栏一共多宽」，不是「内容区多宽」。 所以项目里几乎都会打开 alternative box model。

MDN 推荐写法（注意：box-sizing 默认不继承，所以要让子元素 inherit）： 

html {
  box-sizing: border-box;
}
*, *::before, *::after {
  box-sizing: inherit;
}
