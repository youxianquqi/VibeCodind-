# 用科目一的方式快速入门 Vibe Coding！

学开车先刷科目一，不是上来就上路。  
学 **Vibe Coding**（用 AI 结对写代码）也一样：先把规则考过，再敢放心「飞」。

---

## 一句话

把「能不能瞎用 AI」做成一套**客观题**：判断 / 单选 / 多选，限时交卷，错满就不合格——和科目一同一套肌肉记忆。

---

## 为什么要这样学

| 常见翻车 | 科目一式考点 |
|----------|--------------|
| 密钥、`.env` 丢给模型 | 泄密与隐私 |
| AI 说对了就合并 | 幻觉 / 假验证 |
| 一次改半个仓库 | 改动范围 |
| 盲插 `innerHTML`、假依赖 | 安全与依赖陷阱 |

Vibe Coding 的门槛不是「会不会提问」，而是**会不会拦错误用法**。

---

## 怎么刷（最短路径）

1. **先读路线**：[前端从零开始完整路径](https://github.com/youxianquqi/VibeCodind-)（AI 规范贯穿全程）
2. **综合客观考** `qualifying-exam/`：科目一式多模块，练手感
3. **AI 专项考** `ai-exam/`：每次 20 题、20 分钟，**错 ≥5 不合格**
4. **阶段 0 练习** `stage0-quiz/`：在编辑器里写 TODO，别只在网页里填空

仓库：https://github.com/youxianquqi/VibeCodind-

本地起静态服即可考，例如：

```bash
npx --yes serve ai-exam -l 5178
```

---

## 合格线（记住就行）

- AI 考：对 ≥16 / 错 &lt;5
- 答完一题立即显示对错与解析，到点自动交卷
- 手机、电脑都能答（窄屏用底部答题卡）

---

## 结尾

科目一过了，才有上路资格。  
**Vibe Coding 也是：规则过关，再谈 vibe。**

先开考，再飞。
