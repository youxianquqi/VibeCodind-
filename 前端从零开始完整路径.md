# 前端从零开始完整学习路径

> 7 个阶段，总时长 6-9 个月（兼职）/ 3-4 个月（全职）  
> 配合已有文档：《分阶段问题清单》《需求流程图升级》《时间管理》  
> **2026 修订**：对齐 State of JS / 职业路线共识——地基不变；TS 与框架并行提前；补测试 / 无障碍 / 性能常识；AI 协作写入总纲。主线仍是 **Vue3 + 后台实战（endtiem）**，不改为 React。

## ★ AI 协作规范（贯穿 0–6，2026 必加）

AI（Cursor / Copilot 等）是加速器，不是替身。每个阶段都适用：

### 必须遵守
- **先理解再合并**：看不懂的代码不进主分支 / 不上线
- **先跑再信**：AI 说「已通过」以你本地命令输出为准
- **控范围**：写明相关路径与「不要改无关模块」；大 diff 先审
- **防泄密**：不把 `.env`、密钥、客户 PII、未打码生产数据贴给公共模型
- **安全门禁**：拒绝盲信 `innerHTML` 拼用户输入、假鉴权、乱 CORS、虚构 npm 包

### 配套自测
- AI 专项合格考：[`ai-exam/`](ai-exam/)（抽 20 题，**错 ≥5 不合格**）
- 前端综合客观考（可选）：[`qualifying-exam/`](qualifying-exam/)
- 阶段 0 代码练习：[`stage0-quiz/`](stage0-quiz/)

### 贯穿判据（每阶段自检一句）
能说明：本阶段哪段代码可以让 AI 起草、哪段必须自己写/自己讲清、合并前做了哪几项验证。

## 阶段 0：HTML/CSS + 无障碍入门（1-2 周）

### 学什么
- HTML 标签、语义化（header/nav/main/article/section/footer）
- **无障碍入门**：有意义的 `alt`、`label` 关联表单、合理标题层级（h1–h6）、按钮用 `button` / 链接用 `a`
- CSS 选择器、盒模型、定位
- **Flex 布局**（重点）
- **Grid 布局**（重点）
- 响应式基础（media query、viewport）
- CSS 变量、过渡、动画基础
- （选修，勿抢主线）了解 container queries、`@layer` 名词即可

### 推荐资源
- MDN 入门教程 / MDN 无障碍简介
- freeCodeCamp 响应式网页设计认证
- CSS Flexbox Froggy、CSS Grid Garden
- 练习页：`stage0-quiz/`（先读后写）

### 产出
- 个人主页（静态，含基本无障碍）
- 产品介绍页（响应式）
- 一个纯 CSS 动画作品

### 判据
- 能不查文档写出 Flex 居中、两栏布局、响应式导航
- 能解释 BFC、盒模型、定位的区别
- 图片有合理 alt；表单控件有 label；页面标题层级不乱跳

## 阶段 1：JS 基础（3-4 周）🎮 小游戏起点

### 学什么
**第 1 周：基础语法**
- 变量、类型、运算符
- 条件、循环
- 函数、箭头函数
- 数组/对象方法（map/filter/reduce/find/some/every）

**第 2 周：DOM + 事件** 🎮 能做小游戏
- DOM 选择、操作、增删改
- 事件监听、事件冒泡、事件委托
- 表单事件、键盘事件
- localStorage / sessionStorage
- 警惕：`innerHTML` + 不可信字符串 → XSS 风险（与 AI 规范呼应）

**第 3 周：进阶概念**
- 作用域、闭包（冴羽博客深入系列 01-07）
- this 指向（冴羽博客）
- 原型链、继承（冴羽博客）
- ES6+ 特性（解构、展开、模板字符串、class、模块化）

**第 4 周：异步**
- Promise、async/await
- setTimeout / setInterval
- fetch API
- 事件循环初步（宏任务/微任务）

### 推荐资源
- **冴羽博客深入系列**（必读，阶段 4 二刷）
- MDN JavaScript 教程
- JavaScript.info

### 产出
- 🎮 贪吃蛇（DOM 版）
- 🎮 井字棋（带 AI 对战）
- 🎮 记忆翻牌游戏
- 🎮 2048（学完异步后）
- 🎮 扫雷（学完异步后）

### 判据
- 能不查文档写出贪吃蛇的核心循环
- 能解释闭包、this、原型链（讲给别人听）
- 能用 fetch 调用 API 并渲染数据
- 能说清：哪些逻辑可让 AI 起草，核心循环为何必须自己写得出来

## 阶段 2：工程化 + 最小测试 / 性能（2-3 周）

### 学什么
- **npm / pnpm**（包管理、package.json、lock 文件；AI 改依赖必须人审）
- **Git**（add/commit/push/pull/branch/merge/rebase/conflict）
- **Vite**（dev server、build、ES Modules）——2026 构建体验主流选择
- **ESLint + Prettier**
- 模块化（ESM vs CommonJS）
- 简单的 CI/CD 概念（如 GitHub Actions / Pages）
- **最小测试（P1）**：Vitest 测 1～2 个纯函数（如分数计算、坐标是否出界）
- **性能常识（P1）**：知道 Core Web Vitals 名词；会用 DevTools Network / Lighthouse 看一次自己的页面（不必深钻优化）

### 推荐资源
- Git 官方文档 + Learn Git Branching
- Vite 官方文档、pnpm 官网
- Vitest 官方文档（只读 Getting Started 即可）

### 产出
- 阶段 1 小游戏用 Vite 重构
- 推到 GitHub，配 GitHub Pages（或同类）自动部署
- 至少一个工具函数带 Vitest 用例且本地跑绿
- 对作品集页跑一次 Lighthouse，记下 1～2 个问题（不必全修好）

### 判据
- 能独立用 Vite 起新项目
- 能解决 Git 合并冲突
- 能解释 npm 依赖的 `^` 和 `~`
- 能写并跑通一个最小单测
- 能指出自己页面 Lighthouse 里一项可改进点

## 阶段 3：Vue3 + 后台模板（4-6 周，**新文件优先 TS**）

### 学什么
**第 1-2 周：Vue3 基础**
- 模板语法、指令（v-if/v-for/v-bind/v-on/v-model）
- Composition API（setup、ref、reactive、computed、watch）
- 组件、Props、Emit、defineExpose
- 生命周期

**第 3-4 周：生态**
- Vue Router（路由、守卫、动态路由）
- Pinia（状态管理）
- Axios 封装、拦截器
- Element Plus / Ant Design Vue
- **TS 并行（P0）**：从本阶段起，**新组件 / 新工具函数尽量用 `.ts` / `<script setup lang="ts">`**；类型写不全可以用 `unknown` 推进，禁止堆 `any` 逃避。阶段 5 再系统补深度。

**第 5-6 周：实战项目**
- 学习 **v3-admin-vite**
- 对照改造 **endtiem**（D:\PCCodes\endtiem）
- 表单与按钮保持无障碍底线（label、可聚焦、不要用 div 冒充 button）

**选修了解（勿打乱主线）**
- Nuxt / SSR 适用场景（内容站、SEO）；管理后台继续 SPA + Vite 完全成立
- Tailwind：CSS 扎实后再当提速层，不替代阶段 0

### 推荐资源
- Vue3 官方文档（必读，中文版）
- **v3-admin-vite** 源码
- Pinia 官方文档
- Vue + TS 官方指南（随用随查）

### 产出
- 用 Vue3 重写阶段 1 的一个小游戏
- endtiem：登录 Token + 路由守卫 + 权限菜单 + Axios 封装
- 至少 2～3 个新模块以 TS 书写并可通过类型检查

### 判据
- 能不看 v3-admin-vite，独立给 endtiem 加路由守卫和权限菜单
- 能解释 Vue3 响应式原理（大概，阶段 4 深挖）
- 能用 Pinia 管理跨组件状态
- 新文件默认走 TS，不再「全项目纯 JS 拖到阶段 5」

## 阶段 4：JS 深入 + 框架原理（4-6 周）

### 学什么
**第 1-3 周：JS 深入（冴羽二刷 + 专题）**
- 深入系列二刷（要「能独立写」不是「看懂」）
- 专题：防抖、节流、深拷贝、Promise、Event Loop
- 每篇看完必须自己手写一遍

**第 4-6 周：mini-vue 实现**
- 跟 README 敲一遍 mini-vue
- 响应式（reactive、effect、依赖收集）
- runtime（vdom、diff、挂载、更新）
- 写 3-5 篇原理解析博客

> 2026 说明：AI 会写组件，但**讲清响应式 / 手写基础题**仍是面试与排错溢价。本阶段不要用 AI 代写整份 mini-vue 后假装自己会。

### 推荐资源
- **冴羽博客**（二刷）
- **mini-vue**
- Vue3 源码对照（mini-vue 完成后再看）

### 产出
- 手写题独立完成（防抖/节流/深拷贝/Promise/Event Loop）
- mini-vue 跑通并能讲清每个模块
- 3-5 篇技术博客

### 判据
- 关掉资料，30 分钟内手写 Promise.all、深拷贝
- 能讲清 Vue3 响应式流程
- 能解释为何用 Proxy 而不是 Object.defineProperty

## 阶段 5：TypeScript 深化（2-3 周）

> 阶段 3 已「新文件用 TS」。本阶段目标从「第一次接触」改为 **类型深度 + 端到端重写 + 去掉 any 债**。

### 学什么
- 类型基础查漏（string/number/boolean/array/tuple/enum）
- 接口、类型别名、字面量类型
- 联合类型、交叉类型
- **泛型**（重点）
- 工具类型（Partial/Required/Pick/Omit/Record）
- 类型守卫、类型断言、`unknown` vs `any`
- tsconfig 与 Vue SFC 类型体验
- （选修）type-challenges 简单 + 中等

### 推荐资源
- TypeScript 官方 Handbook
- Vue 官方 TypeScript 指南
- type-challenges

### 产出
- 用 TS **端到端**整理 / 重写 endtiem 关键路径（登录、路由元信息、API 响应类型）
- type-challenges 简单 + 中等合计约 20 题（量力）
- 用泛型封装一个 `useRequest`（或等价）hook

### 判据
- 能解释泛型何时该用、何时过度设计
- 能给 API 响应写出完整类型定义
- 能说明项目里还残留哪些 `any`、为何暂时保留或如何消灭

## 阶段 6：网络/浏览器/算法（持续）

### 学什么
**网络基础**
- HTTP/HTTPS、HTTP/2、HTTP/3
- HTTP 缓存（强缓存、协商缓存）
- 跨域（CORS、代理、JSONP）——结合 AI 规范：警惕「CORS * + 凭证」类危险建议
- Cookie、Session、JWT
- TCP 三次握手、四次挥手（了解）

**浏览器原理**
- 事件循环（宏任务/微任务）
- 渲染管线（DOM → CSSOM → Layout → Paint → Composite）
- 重排重绘
- 垃圾回收
- V8 引擎基础

**算法（前端难度）**
- 数组/字符串、哈希表、双指针、滑动窗口
- 简单 DP
- LeetCode 简单 50 + 中等 20（量力）

### 推荐资源
- 小林 coding（网络图解）
- 浏览器工作原理相关资料
- LeetCode 数组字符串专题

### 产出
- 网络面试题能答约 80%
- 能讲清事件循环与渲染管线
- LeetCode 简单约 50 题

### 判据
- 能讲清「输入 URL 到页面渲染完成」
- 能解释 HTTP 缓存策略
- 能 5 分钟手写一段事件循环执行顺序题

## 重要提醒

1. **关掉资料独立做**——能做出来才算真懂

## 一句话总结

**地基（0–1）→ 小游戏练手 → 工程化 + 最小测试（2）→ Vue 后台实战且新代码用 TS（3）→ 原理与手写（4）→ TS 深化清债（5）→ 网络浏览器算法持续（6）；全程守 AI 协作规范。**

小游戏不是奖励，是练手；AI 不是外挂，是加速器——**能讲清、能验证、能交付**，才算 2026 合格前端学习节奏。
